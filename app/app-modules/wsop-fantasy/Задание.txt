Ежегодно на сайте GipsyTeam проходит конкурс под названием WSOP Fantasy. Суть конкурса заключается в том, что пользователи сайта определенным образом набирают себе команду из игроков Мировой серии покера (World Series Of Poker, WSOP). Цель — собрать команду, которая наберет больше очков WSOP Player of the Year (POY) за время прохождения серии. Естественно, выбор команд проводится до того, как Мировая серия начнется, и очки POY начнут начисляться. К участию в конкурсе допускаются только пользователи, зарегистрированные до публикации конкурса (до 3 июня 2026 года).


Подробные условия выбора команды различаются каждый год. Например, в этом году игроки Мировой серии поделены на 7 групп по 27 игроков (https://docs.google.com/spreadsheets/d/12BMmZ96AEIU6kbGZwL8ldsAusGKdhjHUrZsMH4XMzA4/edit?gid=0#gid=0). Каждому пользователю сайта предлагается собрать одну команду из 7 игроков так, чтобы из каждой группы было ровно по одному игроку. Суммарная стоимость всех 7 игроков не должна превышать 180. Пример валидной команды:

	20
	Ковбои

	55

	Frank Brannan
	25

	27

	Paul Volpe
	5
	Старички

	12
	Старый Свет
	Chris Hunichen
	35
	

Кроме того, одного игрока из команды надо назначить капитаном — для него значение POY будет умножаться на 1,5.


Данные по очкам POY раз в час импортируются с внешнего ресурса. Для каждой команды рассчитывается суммарное значение. На отдельной странице (https://www.gipsyteam.ru/wsop-2026-fantasy, таблица Народный драфт) отображается лидерборд всех команд пользователей сайта с суммарным значением POY и значением POY по каждому игроку в команде и обозначением капитана.


Пользователи сайта хранятся в БД MySQL в таблице:


CREATE TABLE `user` (
  `id` bigint NOT NULL,
  `login` varchar(50) NOT NULL,
  `insert_datetime` datetime DEFAULT NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_0900_ai_ci;
ALTER TABLE `user` ADD PRIMARY KEY (`id`), ADD UNIQUE KEY `login_idx` (`login`);
ALTER TABLE `user` MODIFY `id` bigint NOT NULL AUTO_INCREMENT;


Разбиение игроков на группы производится только для набора команды и никак не используется в дальнейшем.


Требуется разработать оптимальную структуру БД для конкурса, бэкенд, страницу с формой приема команд и страницу для отображения лидерборда. Внешний вид страниц не имеет значения, важны только условия конкурса. Значения очков POY целочисленные, можно взять их случайными для каждого игрока или из колонки Score на странице https://www.25kfantasy.com/players/. Никаких ограничений на инструментарий кроме php v7.4 и MySQL v8.
